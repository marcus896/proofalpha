# Roadmap

## 0.1.x - Public core stabilization

- Validate clean installation on supported Python versions.
- Keep the full test, doctor, package, dependency, and export checks green.
- Improve user-facing quickstarts and evidence-report walkthroughs.
- Document stable schemas, plugin contracts, and migration expectations.
- Expand platform coverage across Windows, Linux, and macOS.

## 0.2.x - Contributor usability

- Add more safe example studies and reproducible fixtures.
- Improve plugin templates and compatibility checks.
- Add clearer benchmark methodology for software performance and reproducibility.
- Resolve high-value legacy type annotation issues.
- Improve troubleshooting and diagnostics.

## 0.3.x - Ecosystem growth

- Publish integration examples.
- Track external contributors, dependent repositories, package downloads, and public references.
- Add privacy-respecting documentation analytics if maintainers approve them.
- Prepare program applications using only current, verifiable adoption evidence.

## Separate approval track

Changes that expand live execution authority are not ordinary roadmap items. They require separate security review, fault testing, risk-policy review, and release-specific disclosure. Paper and no-key behavior remain the public defaults.
